東京管材の社内システム開発の取り組み

TK Lab.

Our In-House AI Initiative

TK Labは、東京管材が社内で進めるシステム開発・AI活用の取り組みです。生成AIを駆使し、私たち自身の手でアプリケーションを内製。70年蓄積した業務の知見をソフトウェアに翻訳しています。汎用ツールではなく、専門商社の現場に最適化された「自分たちのための道具」を自分たちで作り、自分たちで育てています。

THREE LAYERS

3つのレイヤーで、業務を変えています

東京管材のAI活用は、3つの層で進んでいます。「日常業務での生成AI」「自社内製の業務システム」「外部パートナーとの協業」。それぞれが補い合うことで、現場の生産性を抜本的に引き上げます。

// LAYER 01

日常業務での
生成AI活用

社員一人ひとりが、生成AIを日々の業務で活用しています。「ちょっとした作業」を効率化することが、組織全体の生産性向上につながります。

  • メール文面の作成支援
  • 議事録の要約・整理
  • 英文・中国語の翻訳
  • 調査・情報収集
  • 社内資料の作成支援
  • 事務処理の効率化
// LAYER 02 — CORE

自社内製の
業務システム

生成AIを駆使し、専門商社の現場に最適化されたWebアプリケーションを内製。電算室を中心に、現場の声を聞きながら作り、育てています。

  • 内示管理システム
  • 週次集計・自動化ツール
  • 配車最適化システム
  • RPA（業務自動化）の構築
  • クラウド化の推進
// LAYER 03

外部パートナー
との協業

自社で作るより外部パートナーと協業した方が早く・確実に導入できる領域は、外部の力を借ります。「内製と協業」を合理的に使い分けます。

  • AI OCR（注文書データ化）
  • 基幹システムの維持・運用

// LOGISTICS OPTIMIZER

OUR APPROACH

私たちの開発の哲学

ITベンダーに発注して作ってもらうのではなく、自分たちの手で作る。けれど外部の力を借りるべきところは、潔く借りる。「内製」と「協業」を合理的に使い分けながら、現場の声に応えるシステムを育てています。

現場が、自ら作る。

システムを使う人が、システムを作る。営業の流れ、品質保証の手順、物流の動き、それを一番分かっている人が開発に関わる。だから現場に本当に役立つ道具になります。

現場の声を、聞きながら育てる。

「ここが使いにくい」「この機能があったら」。日々の声を反映しながら、システムを継続的に改善。完成形ではなく、生き物として育てる。それが内製の最大の利点です。

内製と協業を、
合理的に使い分ける。

すべて内製にこだわらない。AI OCRのように、信頼できる外部パートナーと協業する方が早く確実な領域は、潔く外注する。「内製主義」ではなく、「合理性」が判断基準です。

70年の経験を、
ソフトウェアに。

創業から70余年、蓄積してきた業務の知見。その判断基準、業務のフロー、現場の暗黙知。それを生成AIの力でソフトウェアに翻訳することが、TK Labの本当の使命です。

// ENGINEERING × AI

技術とAIを
掛け合わせる

営業領域でのAI活用は形になりつつあります。次は品質保証室を中心とした技術領域でも、AIを使いこなしていく。これからの構想を、ここに書いておきます。

// STATUS

これらは現時点で取り組みを開始している、もしくは今後着手していく構想です。実装の進捗に合わせて、内容を更新していきます。

// CONCEPT 01

品質記録の
データ資産化

過去の検査記録、不良対策、サプライヤー別の品質傾向。紙やExcelに散らばっている記録を体系的にデータ化し、AIが扱える資産に変えていきます。70年の経験を「ベテランの頭の中」から「組織の資産」へ。

// CONCEPT 02

技術ノウハウを
AIに教える

鋳造の材質特性、鍛造の温度管理、加工精度の判断基準。蓄積した技術情報を生成AIに読み込ませ、若手社員が必要な時にすぐ参照できる仕組みを構想しています。ベテランに聞かなくても答えが返ってくる環境を、目指しています。

// CONCEPT 03

事務作業から
現場と顧客対応へ

検査報告書の作成、サプライヤーへの問い合わせ、顧客への品質報告。事務的な作業をAIで効率化することで、品質保証室の担当者がより現場や顧客対応に集中できる環境を作っていきます。技術職が、技術に向き合える時間を増やすために。

// DATA SECURITY

情報の保護は最優先事項

AI活用を進めるからこそ、お客様・サプライヤー様からお預かりした情報の保護は最優先事項です。AIに渡してよいデータ、渡さないデータを社内で明確に区別する運用ルールを定め、社員教育を継続的に実施しています。

// POLICY
明文化された
社内ルール

AIに渡してよいデータ、要注意なデータ、絶対に渡さないデータを社内ポリシーで明確に区別。

// CONTRACT
学習対象外の
契約・設定

業務利用するAIサービスは、入力データが学習対象外となる契約・設定で運用。

// EDUCATION
継続的な
社員教育

機密情報の判断基準、仮名化の手順を社員に周知。困った時の相談ルートも整備。
